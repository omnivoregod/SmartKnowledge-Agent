# 🛠️ SmartKnowledge-Agent 实战开发复盘手册

## 📖 项目概述

本项目构建了一个基于 LangChain 1.x (LTS) 架构的智能 Agent。它不仅实现了 RAG（检索增强生成）来感知本地私密文档，还集成了联网搜索、系统时间获取、文件清单管理等工具。项目核心价值在于手动实现了 ReAct 思考循环，并在代码层面解决了轻量级模型在复杂推理中的各种“幻觉”与“惰性”问题。

---

## 🚧 核心困难、障碍及解决方案

### 1. API 接口限制与数据批处理

- **障碍描述**：在构建向量数据库时，若文档切片数量较多（如 252 个片段），调用智谱 AI 的 Embedding 接口会触发 `APIRequestFailedError: input数组最大不得超过64条`。
- **解决思路**：引入分批处理（Batching）逻辑。
- **解决方法**：编写分段加载函数，将切分后的 `docs` 列表按每组 50 个片段进行分割，循环调用 `vector_db.add_documents()`。这保证了海量文档也能稳定初始化。

### 2. LangChain 版本迭代导致的架构断代

- **障碍描述**：项目使用的是最新的 LangChain 1.2.13 (LTS) 版本。该版本将原有的 `AgentExecutor` 和 `create_react_agent` 进行了深度重构甚至移除（不再推荐直接调用），导致大量网络旧教程的导入路径失效。
- **解决思路**：放弃过度封装的“黑盒”工具，转为手动实现 Agent 思考循环。
- **解决方法**：利用 LLM 的 `invoke` 接口，手动编写 `Thought -> Action -> Observation -> Final Answer` 的 `while/for` 循环。这种“白盒化”实现不仅解决了版本报错，还增强了对推理过程的精准控制。

### 3. 轻量级大模型的“模型惰性”与“敷衍式幻觉”

- **障碍描述**：使用 GLM-4-Flash 等轻量级模型时，Agent 常表现出“偷懒”行为。例如：
  - 脑补 Observation：在一次响应中直接伪造工具返回结果。
  - 占位符回答：输出 `(待获取具体内容后填写)` 或 `[...]`。
  - 拒绝服务：幻觉性地回答“由于权限原因，我无法访问 PDF”，即使工具已就绪。
- **解决思路**：采用 **“代码硬约束 + 系统提示词（System Prompt）对齐”** 的混合方案。
- **解决方法**：
  - **逻辑拦截器**：在 Python 代码中检查 Final Answer。若包含省略号、中括号或在未调用 `knowledge_base` 的情况下回答“无法访问”，则强行打回并注入“系统警告”Observation。
  - **物理打断**：在解析 Action 时，若模型在一次输出中脑补了 Observation，代码通过字符串分割仅保留 Action 之前的内容，强制模型等待真实系统反馈。

### 4. RAG 检索的“碎片化”与“时序混乱”

- **障碍描述**：当用户要求“总结全文”或“简述摘要”时，传统的向量搜索仅返回相似度最高的碎片（Top-K），导致 Agent “只见树木不见森林”，且检索出的片段往往逻辑颠倒。
- **解决思路**：引入 **“意图感知检索”与“物理顺序重排”**。
- **解决方法**：
  - **动态 K 值**：在检索函数中检测关键词（如“总结”、“摘要”），自动将 k 从 5 调高到 10。
  - **时序重排**：利用 Chroma 存储的元数据 `metadata`，根据 `page` 页码对检索结果进行物理升序排列，确保交给 LLM 的是逻辑通顺的“连续文本”。

### 5. 时间感知缺失

- **障碍描述**：LLM 没有内置时钟，且联网搜索（Search Tool）常返回过时的网页摘要，导致 Agent 在回答“今天几号”时产生严重幻觉（如回答 2023 年）。
- **解决思路**：赋予 Agent **“原生时钟工具”**。
- **解决方法**：定义一个简单的 `get_current_time` 工具，直接调用 Python 的 `datetime` 库获取物理系统时间。并在 System Prompt 中规定涉及日期时必须优先使用此工具。

## 🏗️ 架构演进：从“面条代码”到“解耦设计”

在项目后期，我们进行了深度的工程化重构：

| 模块名称 | 职责 | 核心代码 |
|----------|------|----------|
| `core/knowledge_base.py` | 数据层 | 负责 PDF 加载、Overlap 分片、Chroma 持久化 |
| `core/prompts.py` | 指令层 | 存放通用的 ReAct 推理协议与搜索策略 |
| `core/agent_logic.py` | 控制层 | 实现正则解析、拦截器逻辑、自省循环 |
| `tools/tool_definitions.py` | 执行层 | 封装具体的 Tool 对象与映射表 |
| `main.py` | 入口层 | 负责环境初始化与用户交互循环 |
